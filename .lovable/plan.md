

## Market Breadth Report — Add Real Paper Trading P&L Data

### Confirmation

You are correct: the Market Breadth Report currently does **not** use any real paper trading data. It is a general market analysis report generated by AI with no user-specific portfolio information. The P&L/balance shown in the **morning briefing email** is separate and does use real data.

### What Changes

**1. Fetch user's paper trading data on the frontend** (MarketBreadthReport.tsx)
- On page load (if user is authenticated), query `paper_portfolios` for `current_balance` and `total_pnl`
- Query recent `paper_trades` (closed today/this week) for a performance snapshot
- Display a "Your Portfolio" summary card above or alongside the market report showing: account balance, total P&L, today's P&L, open positions count, and win rate

**2. Pass user portfolio context to the report generator** (get-cached-market-report edge function)
- Accept optional `portfolioContext` in the request body (balance, P&L, open positions)
- When present, append a personalized section to the AI prompt asking it to contextualize market movements relative to the user's positions
- The cached report remains shared/generic; the portfolio overlay is rendered client-side so caching is not broken

**3. Client-side portfolio overlay card**
- New component `PortfolioSummaryCard` rendered above the report content
- Shows: current balance, total P&L (colored), today's closed trade P&L, number of open positions
- Data fetched directly from Supabase on the client — no edge function changes needed for this part
- Only visible when user is authenticated

### Technical Details

**Files modified:**
- `src/pages/MarketBreadthReport.tsx` — add `usePaperTrading` hook or direct Supabase queries for portfolio data; render portfolio summary card
- New component `src/components/report/PortfolioSummaryCard.tsx` — displays real balance, P&L, open trades
- `src/i18n/locales/en.json` — add translation keys for portfolio summary labels

**No database or edge function changes required.** All data already exists in `paper_portfolios` and `paper_trades` tables. The existing `usePaperTrading` hook can be reused.

### Approach
- Keep the market report itself as a shared cached resource (no per-user generation)
- Layer the user's real portfolio data on top as a separate UI section
- This avoids breaking the caching/budget system while giving each user their real P&L context

## PERMANENT RULE — i18n / Internationalisation (MUST FOLLOW ON EVERY CHANGE)

> This rule applies to ALL code changes in this repository without
> exception. ChartingPath serves users in 18 languages. Every visible
> string MUST be translatable.

### The Rule

EVERY user-visible string in JSX must be wrapped with
`t('namespace.key', 'English fallback')`.

No exceptions for:
- Labels, headings, descriptions, tooltips
- Button text, link text, badge text
- Error messages, empty states, loading states
- Metric labels ("Win rate", "Total detections", etc.)
- Dynamic strings with interpolation:
  t('key', 'text {{var}}', { var: value })
- PageMeta title and description props
- document.title assignments

### How to do it correctly

Step 1 — Namespace keys by component:
- Landing components → landing.keyName
- Pattern library → patternLibrary.keyName
- FAQ → faq.tabName.keyName
- Blog → blog.keyName
- Pricing → pricingPage.keyName or pricingTeaser.keyName
- Outcome stats → outcomeStats.keyName
- About → aboutPage2.keyName
- Shared → common.keyName

Step 2 — Use t() in JSX:
```tsx
// WRONG:
<div>Win rate</div>

// CORRECT:
<div>{t('patternLibrary.winRate', 'Win rate')}</div>
```

Step 3 — Add every new key to src/i18n/locales/en.json.
The auto-translation system handles all other 17 languages
automatically, but only if the key exists in en.json first.

Step 4 — Import useTranslation in every component:
```tsx
import { useTranslation } from 'react-i18next';
const { t } = useTranslation();
```

### PageMeta special case

```tsx
// WRONG:
<PageMeta title="Simple Pricing." description="Start free..." />

// CORRECT:
<PageMeta
  title={t('pricingPage.metaTitle', 'Simple Pricing. | ChartingPath')}
  description={t('pricingPage.metaDescription', 'Start free...')}
/>
```

### Checklist before completing any task

- [ ] Every visible string uses t('namespace.key', 'fallback')
- [ ] Every new key is added to src/i18n/locales/en.json
- [ ] useTranslation imported in every component with text
- [ ] Dynamic strings use interpolation: t('k', 'text {{v}}', {v})
- [ ] PageMeta title and description use t()
- [ ] No raw string literals in JSX (no >SomeText< patterns)
